手写一个plugin

发表于2024-05-24|更新于2024-05-24
|字数总计:729|阅读时长:2分钟|阅读量:

plugin 提供了很多比 loader 中更完备的功能，他使用阶段式的构建回调，webpack 给我们提供了非常多的 hooks 用来在构建的阶段让开发者自由的去引入自己的行为。

基本结构

一个最基本的 plugin 需要包含这些部分：

  • 一个 JavaScript 类
  • 一个 apply 方法，apply 方法在 webpack 装载这个插件的时候被调用，并且会传入 compiler 对象。
  • 使用不同的 hooks 来指定自己需要发生的处理行为
  • 在异步调用时最后需要调用 webpack 提供给我们的 callback 或者通过 Promise 的方式（后续异步编译部分会详细说）

Compiler 和 Compilation

区别
Compiler 为一个和 webpack 环境整体绑定的一个对象包括 options，loader 和 plugin。
当 webpack 启动时，这个对象会被实例化，并且他是全局唯一的，apply方法的入参。
Compiler 代表整个 Webpack 编译过程的实例。<br /
它负责创建 Compilation 实例，并在 Webpack 生命周期中起着主要的控制作用。
Compiler 实例是全局唯一的，贯穿整个编译过程，从 Webpack 启动到关闭。 它管理插件，控制整个编译过程的执行，可以注册和执行插件，监听 Webpack 的事件。
Compiler 不包含具体的编译信息，只负责启动和控制编译过程。
Compilation Compilation 代表一次具体的编译过程的实例。
每次编译都会创建一个新的 Compilation 实例。
它封装了当前编译的所有信息，包括当前编译的所有模块、资源和编译生成的输出文件等信息。
Compilation 提供给 loader 和插件使用，可以获取当前编译的资源、模块、依赖关系等信息，并进行处理。
